Uso semântico de botões

Os botões (elemento <button>) devem ser utilizados para acionar uma funcionalidade e seu propósito deve ser claramente identificável. Ou seja, os usuários, inclusive os que utilizam leitor de tela ou outras tecnologias assistivas, devem compreender facilmente qual o propósito do botão, que funcionalidade ele irá acionar. Assim, o texto do botão deve ser descritivo, refletindo a ação que ele desencadeia. Deve-se evitar, portanto, textos genéricos como "Clique aqui", preferindo descrições claras como "Enviar formulário" ou "Excluir item".

É importante salientar que o uso de elemento HTML apropriado para botões é crucial. O elemento <button> é preferível ao <div> ou <span> estilizados como botões, pois o <button> tem valor semântico e é automaticamente reconhecido como botão pelas tecnologias assistivas.

Exemplo de botões descritivos

A seguir, temos o formulário correspondente à última etapa de uma compra online. Nessa etapa, o usuário informa os dados do cartão de crédito para efetuar o pagamento. O formulário possui dois botões, um para pagar a compra e outro para limpar os campos, caso o usuário deseje por ter preenchido incorretamente.

Repare que os textos dos botões,"Enviar Formulário" e "Cancelar", são claros e descrevem a ação que cada botão realizará.

O <button type="submit">, utilizado para enviar o formulário, é semântico e acessível, indicando claramente que o botão envia o formulário. Já o <button type="button">, utilizado para cancelar a ação atual e redirecionar o usuário para outra página, garante que o botão não envie o formulário acidentalmente. Outra característica importante é que os botões têm estilos distintos (submit-button e cancel-button), o que ajuda a diferenciar suas funções e a tornar a interface mais intuitiva.
