Critério 1.3.6

Codificação e Marcação Semântica

13. Propósito identificável: A finalidade dos componentes da interface do usuário, ícones e regiões pode ser determinada programaticamente.

O critério 1.3.6 das WCAG estabelece que a finalidade dos componentes da interface do usuário, como botões, ícones e regiões, deve ser determinada programaticamente. Isso é fundamental para garantir que usuários de tecnologias assistivas, como leitores de tela, possam compreender e interagir corretamente com esses elementos.

Para que a finalidade dos componentes seja determinada, é essencial o uso correto de elementos HTML semânticos e atributos ARIA, que fornecem informações adicionais sobre a função de um componente. Esses atributos ajudam a transmitir a função e a estrutura dos elementos, permitindo uma navegação mais intuitiva e acessível.

Explicação do Exemplo:

Abaixo um exemplo de um formulário de contato que utiliza elementos HTML semânticos, atributos ARIA e ícones SVG, garantindo que os leitores de tela possam entender claramente a finalidade de cada componente:

  • Estrutura Semântica:

    O uso de elementos semânticos como <h1>, <h2>, <label>, <input> e <textarea> ajuda os leitores de tela a entenderem a hierarquia e a estrutura do conteúdo.

  • Atributo aria-labelledby:

    O atributo aria-labelledby no formulário vincula a descrição do formulário ao título, ajudando os leitores de tela a identificar o propósito do formulário.

  • Atributos aria-describedby:

    Cada campo de entrada tem um atributo aria-describedby, que aponta para um elemento <span> que fornece informações adicionais sobre o que é esperado nesse campo.

  • Atributo aria-label:

    O botão de envio possui um atributo aria-label que fornece uma descrição clara de sua finalidade, permitindo que os usuários de leitores de tela compreendam a ação.

  • Ícone SVG:

    Um ícone gráfico (SVG) é incluído no botão para melhorar a funcionalidade visual, mas não substitui a necessidade de uma descrição textual clara. O ícone é visualmente acessível, mas a descrição do botão é fornecida pelo atributo aria-label.

  • Classe .visually-hidden:

    A classe .visually-hidden foi adicionada ao estilo para ocultar visualmente elementos, mas ainda torná-los acessíveis para leitores de tela.
