Critério 1.4.1

Uso de Cores

1. Uso de cores: Não há elementos que fazem uso somente de cores para transmitir informações, indicar uma ação, pedir uma resposta ou distinguir um elemento visual.

O critério 1.4.1 das WCAG estabelece que não deve haver elementos que façam uso somente de cores para transmitir informações, indicar ações, pedir respostas ou distinguir elementos visuais. Isso é importante para garantir que todos os usuários, incluindo aqueles com deficiências visuais como daltonismo, possam acessar e entender o conteúdo.

É essencial fornecer informações de forma que não dependam exclusivamente da percepção de cor. Para isso, é recomendado o uso de texto, ícones ou outros elementos visuais que possam transmitir a mesma informação que a cor sozinha.

Explicação do Exemplo:

No exemplo abaixo, as mensagens informativas e de erro são representadas não apenas por cores, mas também por texto e ícones. A mensagem informativa possui um fundo azul claro e texto azul, enquanto a mensagem de erro tem um fundo vermelho claro e texto vermelho. Ambos os casos incluem texto descritivo que fornece informações essenciais sobre a ação ou status, tornando o conteúdo acessível para todos os usuários.

  • Informação Visual e Textual: As mensagens de informação e erro são apresentadas em diferentes cores, mas também incluem texto que explica a natureza da mensagem. Isso garante que os usuários que não conseguem distinguir cores ainda possam entender a mensagem.
  • Estilos CSS: As mensagens têm estilos diferenciados, usando cores de fundo e texto que chamam a atenção, mas a informação essencial é transmitida por meio do texto descritivo, cumprindo assim o especificado no critério 1.4.1.
  • Acessibilidade: O uso de texto descritivo e ícones garante que a informação não dependa apenas da cor, tornando a interface mais acessível para usuários com deficiências visuais.
