Critério 1.4.11

Uso de Cores

4. Contraste para componentes: Todo componente de interface tem relação de contraste de pelo menos 3:1 com o fundo e entre seus estados.

O critério de sucesso 1.4.11 da WCAG exige que componentes de interface (como botões, links, controles, etc.) tenham uma relação de contraste de pelo menos 3:1 em relação ao fundo. Isso inclui estados dinâmicos, como hover, foco e pressionado, garantindo que todos os usuários possam identificar visualmente os componentes e suas mudanças de estado.

No exemplo abaixo, um botão foi estilizado para atender a essa recomendação. A relação de contraste entre o botão e seu fundo é de pelo menos 3:1 em todos os estados. Além disso, a transição suave ajuda a destacar as mudanças de estado.

Explicação do Exemplo:

  • Cores e Contraste: As cores escolhidas garantem uma relação de contraste de pelo menos 3:1 entre o fundo, o botão e sua borda.
  • Estados: O estado hover foi estilizado para indicar visualmente a interação com contraste adequado.
  • Transição: A propriedade transition melhora a experiência do usuário ao tornar as mudanças mais perceptíveis.
  • Estado pressionado: No estado :active utiliza uma tonalidade de azul mais escura para o fundo e uma borda ainda mais escura, garantindo contraste de pelo menos 3:1 com o texto branco e o fundo.
  • Feedback tátil visual: O uso de transform: scale(0.98) cria uma leve compressão, ajudando o usuário a identificar que o botão foi pressionado.
