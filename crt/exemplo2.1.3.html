Critério 2.1.3

Interação por teclado

5. Uso de foco: Todos os elementos focáveis são utilizados para interação.

O critério 2.1.3 da WCAG estabelece que todos os elementos focáveis de uma página devem ser úteis e proporcionarem interação ao usuário. Elementos que podem ser focados, como links e botões, não devem ser vazios ou decorativos, garantindo que o foco do teclado seja utilizado apenas onde há função ou interação.

Este princípio evita que o usuário perca tempo navegando em elementos focáveis que não possuem propósito ou ação, melhorando a eficiência e a experiência de navegação para usuários que dependem do teclado.

Explicação do Exemplo:

  • Uso Significativo do Foco:
    • Apenas elementos com função (botões e links com ações claras) podem ser focados.
    • Elementos decorativos ou sem função não são focáveis.
  • Destaque ao Foco:
    • O foco é visualmente destacado quando o usuário navega com o teclado, utilizando bordas e sombras.
  • Elementos Interativos:
    • O botão e o link possuem funções úteis (exemplo: exibir alertas), garantindo que o usuário interaja apenas com elementos funcionais.
