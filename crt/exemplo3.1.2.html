Critério 3.1.2

Codificação e Marcação Semântica

3. Idioma das partes da página: Todas as partes da página que possuem idioma diferente do principal têm seu idioma declarado de forma programaticamente determinada ou o texto é um nome próprio, um termo técnico, uma palavra em um idioma indeterminado, ou uma palavra ou frase estrangeira que se tornou parte do vernáculo do texto que a envolve.

O critério 3.1.2 das WCAG exige que partes de uma página da web cujo idioma seja diferente do idioma principal sejam declaradas programaticamente, ou seja, estejam especificadas no código fonte do HTML. Isso permite que tecnologias assistivas, como leitores de tela, ajustem a pronúncia corretamente para cada idioma específico, garantindo uma melhor compreensão para o usuário. Isso é essencial para frases estrangeiras, termos técnicos ou citações em outros idiomas, garantindo que os usuários entendam corretamente o conteúdo.

Essa técnica deve ser utilizada quando forem:

  • Usadas palavras ou frases estrangeiras que não fazem parte do vernáculo local.
  • Citados textos em outro idioma.
  • Incluidos termos técnicos em outro idioma.

Considere o trecho abaixo onde parte do texto está em inglês, enquanto o restante está em português:

Explicação do Exemplo:

  • A tag <html> inclui o atributo lang="pt-BR", especificando que o idioma principal da página é o Português do Brasil.
  • O termo "Web Accessibility" está em inglês e, por isso, é envolvido pela tag <span> com o atributo lang="en".
  • Essa declaração permite que leitores de tela ajustem a pronúncia corretamente ao ler essa frase específica em inglês.

Termos estrangeiros que se tornaram parte do idioma local (por exemplo, "e-mail" ou "download") não precisam de uma declaração especial, pois são geralmente compreendidos como parte do vernáculo.

<!DOCTYPE html>
	<html lang="pt-BR">
	<head>
		<meta charset="UTF-8">
		<title>Exemplo com Idiomas Diferentes</title>
	</head>
	<body>
		<p>Este site segue as diretrizes de acessibilidade da WCAG.
		O termo <span lang="en">Web Accessibility</span> é amplamente utilizado para descrever
		práticas de acessibilidade na web.</p>
	</body>
	</html>
