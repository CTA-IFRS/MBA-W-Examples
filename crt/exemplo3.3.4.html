Critério 3.3.4

Formulários e Entrada de Dados

13. Ajuda contextual: Há ajuda contextual, como instruções e dicas relevantes, que auxilia o usuário no preenchimento e envio dos formulários.

O critério 3.3.4 da WCAG assegura que os formulários ofereçam instruções e dicas relevantes para facilitar o preenchimento correto pelos usuários. Essas instruções devem ser:

  • Disponíveis no momento apropriado (antes ou durante o preenchimento);
  • Acessíveis a todos os usuários, incluindo aqueles que utilizam tecnologias assistivas;
  • Claras e fáceis de entender, para evitar erros e confusões.

O exemplo abaixo implementa um formulário com ajuda contextual clara e acessível, utilizando instruções relevantes e disponíveis no momento certo.

Explicação do Exemplo:

Os elementos do formulário e do script, trazidos no exemplo, que contribuem para atender ao critério 3.3.4 são:

  • Elementos do Formulário
    • aria-describedby: Cada campo possui este atributo associado ao ID de um elemento <small> que fornece ajuda contextual específica.
      Exemplo: O campo de nome está vinculado à mensagem "Insira seu nome completo conforme consta em seus documentos oficiais." Isso garante que leitores de tela associem corretamente o campo ao texto de ajuda.
    • aria-hidden="true": Os asteriscos usados para indicar campos obrigatórios estão marcados como aria-hidden="true", garantindo que leitores de tela não interpretem os asteriscos como conteúdo adicional irrelevante.
    • Mensagens de ajuda claras e específicas: Cada campo tem uma descrição significativa que orienta os usuários.
      Exemplo para o campo de e-mail: "Digite um e-mail válido. Exemplo: usuario@dominio.com."
  • Comportamento Dinâmico (Script de Validação)
    • Foco nas instruções: Quando um campo recebe foco, a mensagem de ajuda associada é destacada visualmente (cor azul), chamando a atenção do usuário. Isso beneficia usuários com dificuldades de concentração ou visão reduzida.
    • Retorno ao estado padrão: Quando o campo perde o foco, o destaque visual é removido, garantindo uma interface limpa e intuitiva.
  • Benefícios para Acessibilidade
    • Ajuda contextual fornecida de maneira previsível, próxima aos campos correspondentes, beneficia tanto usuários visuais quanto aqueles que utilizam tecnologias assistivas.
    • Implementações como aria-describedby e comportamento dinâmico tornam o formulário acessível para diferentes grupos, incluindo usuários com deficiência visual ou cognitiva.
