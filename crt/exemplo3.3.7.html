Critério 3.3.7

Formulários e Entrada de Dados

8. Reentrada de dados: "Todos os campos de entrada que exigem dados previamente inseridos são preenchidos automaticamente ou estão disponíveis para seleção ou a reentrada de dados é essencial ou a informação inserida é necessária por segurança ou a informação previamente inserida não é mais válida."

O critério 3.3.7 da WCAG exige que os formulários ofereçam uma forma de minimizar a necessidade de reentrada de dados. Isso significa que, quando possível, os campos de entrada que exigem dados previamente inseridos devem ser preenchidos automaticamente ou fornecer opções para que o usuário selecione ou confirme os dados, sem que seja necessário preenchê-los novamente. A reentrada de dados só é aceitável quando é essencial, como por questões de segurança, ou quando a informação inserida previamente não é mais válida.

Este critério é importante para melhorar a experiência do usuário, especialmente para aqueles que usam tecnologias assistivas ou têm dificuldades motoras ou cognitivas. A ideia é evitar que o usuário precise inserir os mesmos dados várias vezes, o que pode ser cansativo, propenso a erros e frustrante.

O exemplo abaixo apresenta um formulário de cadastro de conta com campos para nome, email, senha e confirmação de senha.

Explicação do Exemplo:

  • O valor do campo "Nome" e "Email" foi preenchido automaticamente com informações previamente inseridas. Isso minimiza a necessidade de reentrada de dados, conforme recomendado pela WCAG.
  • value="João Silva" e value="joao.silva@example.com" preenchem esses campos com valores já conhecidos ou que o usuário forneceu em um cadastro anterior.
  • O campo de senha e confirmação de senha exige que o usuário insira a senha manualmente por questões de segurança, o que está em conformidade com a recomendação, pois a reentrada é necessária em razão da segurança.
  • O formulário permite que os dados sejam enviados diretamente após o preenchimento, sem a necessidade de reentrar com dados que já foram inseridos e validados anteriormente, exceto para a senha, onde a reentrada é necessária para garantir que ambas as senhas coincidam.
