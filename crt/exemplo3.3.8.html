Critério 3.3.8

Formulários e Entrada de Dados

18. Autenticação acessível (mínima): Não há testes de função cognitiva em processos de autenticação ou há um método de autenticação alternativo sem teste de função cognitiva ou há um mecanismo para auxiliar o usuário a completar o teste de função cognitiva ou o teste é um método de reconhecimento de objetos ou o teste serve para identificar conteúdo não textual fornecido pelo usuário.

De acordo com o 3.3.8 da WCAG, não é permitido exigir testes de função cognitiva em processos de autenticação. Caso testes cognitivos sejam exigidos, deve existir um método alternativo de autenticação ou um mecanismo para auxiliar o usuário a completar o teste. Além disso, a autenticação deve ser realizada de forma acessível, sem sobrecarregar cognitivamente o usuário.

O critério também permite que o método de autenticação utilize o reconhecimento de objetos ou seja utilizado para identificar conteúdo não textual fornecido pelo usuário. Dessa forma, os testes devem ser desenhados para não criar barreiras àqueles com dificuldades cognitivas.

Explicação do Exemplo:

Este exemplo evita testes cognitivos complexos, oferecendo uma autenticação alternativa baseada em um CAPTCHA visual simples. Além disso, oferece o uso de uma autenticação por imagem (método de reconhecimento de objetos), uma abordagem acessível para usuários que podem ter dificuldades cognitivas.
