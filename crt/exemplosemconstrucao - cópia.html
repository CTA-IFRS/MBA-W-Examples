Formulários e Entrada de Dados

1 - Campos com rótulos descritivos

Todos os campos de entrada precisam ter um rótulo descritivo associado a ele, por exemplo, usando a tag <label>.

Isso é crucial para garantir que usuários com deficiências, especialmente aqueles que utilizam tecnologias assistivas como leitores de tela, possam entender o propósito de cada campo do formulário de forma clara.

Exemplo

Abaixo o exemplo de um formulário para envio de uma mensagem:

Cada campo de entrada do formulário tem um rótulo associado através da tag <label>. Por exemplo, o campo de nome tem um <label> com o texto "Nome completo" associado ao campo de texto com o atributo for="nome".

A tag <label> ajuda os usuários de leitores de tela a compreenderem rapidamente o propósito do campo. Sem esse rótulo, eles podem ter dificuldade em entender o que deve ser preenchido.

O atributo for do <label> está vinculado ao atributo id do campo de entrada correspondente. Isso cria uma conexão explícita entre o rótulo e o campo, o que é essencial para a acessibilidade, pois permite que tecnologias assistivas reconheçam essa relação.

Exemplo: <label for="nome">Nome completo:</label> e <input type="text" id="nome" name="nome">.

Observe quetanto o atributo for como o id tem a mesma identificação "nome", criando o vínculo entre os dois campos.

Usuários que dependem de leitores de tela ou navegam apenas pelo teclado podem ouvir ou ver o rótulo sendo lido ao focar no campo do formulário, permitindo que eles saibam exatamente o que precisa ser inserido. Sem essa associação, seria impossível para eles entenderem o propósito do campo.

2 - Agrupamento de campos relacionados

Quando temos campos de entrada que são relacionados entre si, como um grupo de opções de um formulário (por exemplo, informações de endereço), é importante agrupá-los usando elementos como <fieldset> e <legend>. Esses elementos ajudam tanto na organização visual quanto na acessibilidade do formulário, especialmente para usuários que utilizam tecnologias assistivas, como leitores de tela.

A utilização de elementos como <fieldset> e <legend> auxilia a fornecer contexto e estrutura clara para grupos de controles de formulário relacionados. Isso melhora a experiência de navegação e interpretação de um formulário, especialmente para pessoas com deficiência visual.

Exemplo:

Abaixo o exemplo de um formulário para envio de uma mensagem onde campos relacionados foram agrupados:

O <fieldset> é um elemento HTML que agrupa vários controles (campos de entrada) e rótulos (<label>) dentro de um formulário. Isso permite criar uma separação visual e semântica para um conjunto de campos relacionados.

No exemplo, há dois grupos de campos:

  • Um grupo de "Dados Pessoais" (nome, e-mail, telefone).
  • Um grupo de "Mensagem" (campo para a mensagem do usuário).

Cada conjunto de campos é envolvido por um <fieldset>, criando uma distinção clara para usuários e leitores de tela.

O <legend> é o rótulo que descreve o propósito do grupo de campos dentro de um <fieldset>. Ele age como um "título" para o grupo.

No exemplo, os <legend> são "Dados Pessoais" e "Mensagem". Esses rótulos ajudam a identificar rapidamente o que aquele conjunto de campos representa.

Usuários que utilizam tecnologias assistivas, como leitores de tela, se beneficiarão do uso de <fieldset> e <legend>, pois os leitores de tela anunciam o conteúdo de cada <legend>, deixando claro qual é o propósito do grupo de campos de formulário.

Isso ajuda a navegar de forma eficiente por formulários complexos, garantindo que o usuário compreenda melhor a relação entre os campos e suas funções.

Links e Navegação

1 - Todos os links são utilizados para navegação

Deve-se garantir que todos os links da página sejam usados unica e exclusivamente para navegação, seja ela interna (navegação entre ancoras nas seções da mesma página) ou exeterna (para outros sites).

Adicionalmente é recomendado que:

  • Todos os links devem ter textos descritivos que informem o usuário sobre o destino do link. Evitando usar textos genéricos como "clique aqui" ou "saiba mais", pois não são informativos para usuários de tecnologias assistivas.
  • Links que permitem a navegação dentro da mesma página (âncoras) devem ser claramente identificáveis e devem ter um propósito descritivo.
  • Links para outros sites devem deixar claro que o usuário está saindo do site atual. Isso pode ser feito com textos descritivos ou indicando que o link abrirá em uma nova janela ou aba.

Como inserir texto alternativo para imagens em blogs e redes sociais

A descrição de imagens é uma prática muito importante para que pessoas com deficiência visual ou com conexões lentas de internet possam ter acesso às informações que aparecem nesses elementos. A descrição pode ser disponibilizada por meio do texto alternativo, também conhecido como “alt”, por meio de uma legenda ou, ainda, no próprio conteúdo que antecede ou sucede a imagem, ou seja, no contexto.

Diversos softwares, como editores de texto, oferecem opção para fornecer um texto alternativo para as imagens. Mais recentemente, gerenciadores de conteúdo e redes sociais também passaram a contemplar essa funcionalidade. Abaixo, aprenderemos como inserir o texto alternativo nas principais plataformas de blogs e redes sociais, sendo elas:

  • Wordpress
  • Facebook
  • Instagram
  • LinkedIn
  • Pinterest

Wordpress

Para inserir um texto alternativo em uma imagem no WordPress, após inseri-la na postagem, basta clicar sobre ela e, em seguida, clicar na opção de editar (botão com ilustração de lápis). Na sequência, na janela “Detalhes da imagem”, insira a descrição no campo “Texto alternativo”. Para finalizar, clique no botão “Atualizar” no final da janela e pronto, a descrição será salva.

Facebook

Para inserir o texto alternativo no Facebook, basta clicar no “Feed de Notícias” , inserir a imagem na opção “Foto/Vídeo”, clicar na opção “Editar”, clicar na opção “Texto alternativo”. O Facebook já fornece uma sugestão de descrição, porém se quiser adicionar um texto personalizado, clique em “Texto alternativo personalizado” e insira a descrição. Para encerrar, pressione o botão “Gravar”. Para mais detalhes, acesse a postagem do CTA como inserir texto alternativo para imagens no Facebook.

Instagram

No instagram, adicione a imagem, realize a edição com filtros e na tela para publicação clique em “Configurações avançadas”. Na opção “Acessibilidade” haverá um campo para inserção do texto alternativo.

LinkedIn

No LinkedIn, após inserir a imagem, clique na opção “Texto alternativo” (logo abaixo da figura). Na janela seguinte, digite a descrição da imagem no campo “Texto alternativo” e, para finalizar, clique no botão “Concluído ”. Pronto, agora é só publicar o conteúdo.

Pinterest

No Pinterest, insira um novo Pin/imagem ao clicar no botão “+”, “Criar um Pin” (canto inferior direito da tela), insira a imagem e clique no botão “Adicionar texto Alternativo”. Será aberto um campo, onde será permitido inserir a descrição. Ao finalizar, clique em “Salvar”.

Botões e Controles

1 - Todos os botões devem ser utilizados para acionar uma funcionalidade.

Todos os botões presentes numa página web devem ser claramente identificáveis e suas funcionalidades devem ser explícitamente compreensíveis para que todos os usuários, incluindo aqueles que utilizam leitores de tela e outras tecnologias assistivas, possam entender o que o botão faz

O uso de elementos HTML apropriados para botões é crucial. O elemento <button> é preferível ao <div> ou <span> estilizados como botões, pois o <button> é semanticamente correto e é automaticamente reconhecido por tecnologias assistivas.

Listas

Todas as listas são utilizadas para agrupamento de itens de mesma natureza.

As listas devem ser usadas de maneira semântica e apropriada para garantir que a estrutura do conteúdo seja clara e compreendida por todos os usuários, incluindo aqueles que usam leitores de tela.

Sendo assim, listas são utilizadas para agrupar itens que compartilham uma mesma natureza ou são partes de um conjunto. Por exemplo, uma lista de recursos, uma lista de passos em um procedimento, ou uma lista de itens de menu.

O uso de elementos HTML semânticos apropriados para listas é fundamental. O elemento <ul> (lista não ordenada) é usado para itens sem uma ordem específica, enquanto <ol> (lista ordenada) é utilizado quando a sequência dos itens é importante. Os itens individuais são definidos com <li> (list item).

Deve-se evitar a utilização de números para identificar uma lista não ordenada, já que isso contradiz a semântica programática do conteúdo. Para esses casos é recomendável utilizar o elemento semantico correto (<ol>)

Também não é uma boa prática crar uma lista onde cada item desta lista é uma outra lista de apenas um item. De forma a respeitar a sememântica dos elementos HTML deve ser usar o <li> para idetificar cada item da lista individualmente.

Listas bem estruturadas ajudam leitores de tela a comunicar a relação entre itens, facilitando a navegação e compreensão do conteúdo. Isso é importante para garantir que todos os usuários possam entender o agrupamento e a ordem dos itens listados.

Conteúdo Textual

Todos os trechos de texto de ênfase, citação, abreviação ou outro tipo de texto especial utilizam elementos específicos para transmitir esses propósitos.

As informações e relacionamentos em uma página web devem ser programaticamente identificáveis e corretamente apresentados. Nesse sentido, todos os trechos de texto que expressam ênfase, citação, abreviação ou outro tipo de informação especial devem ser marcados semanticamente de acordo com seu propósito para que sejam acessíveis por tecnologias assistivas, como leitores de tela.

Elementos como <strong>, <abbr>, <cite>, <blockquote>, <q>, entre outros semelhantes, fornecem contexto adicional para tecnologias assistivas, permitindo que os leitores de tela anunciem corretamente a importância de uma ênfase, o significado de uma abreviação ou o propósito de uma citação.

Tabelas

Todas as tabelas são utilizadas para apresentar dados tabulares. Não há tabelas de leiaute com semântica determinada.

De acordo com as recomendações da WCAG (Web Content Accessibility Guidelines) e as boas práticas de acessibilidade na web, tabelas devem ser utilizadas exclusivamente para a apresentação de dados tabulares — ou seja, dados que são organizados em colunas e linhas e que têm relação entre si. O uso de tabelas para a disposição ou layout visual de elementos na página é desencorajado, pois isso pode causar sérios problemas de acessibilidade.

Quando uma tabela é usada corretamente para apresentar dados, tecnologias assistivas, como leitores de tela, podem processar corretamente as relações entre os dados e suas descrições, permitindo que usuários com deficiência visual compreendam o conteúdo de forma eficaz. Para garantir isso, é essencial usar a marcação semântica dos elementos HTML em tabelas de forma correta, conforme abaixo:

  • Cabeçalhos de Tabela (<th>): Cabeçalhos de linhas e colunas devem ser marcados corretamente com a tag <th>. Isso ajuda os leitores de tela a identificar o propósito de cada célula.
  • Atributo scope: O uso de scope="col" e scope="row" nas células de cabeçalho (headings) ajuda a vincular cada célula de dados à célula de cabeçalho correspondente.
  • Elementos de agrupamento (<caption>, <thead>, <tbody>, <tfoot>): Esses elementos permitem agrupar e organizar a estrutura da tabela, facilitando a compreensão para todos os usuários.
  • Tabelas nunca devem ser usadas para layout de página: Isso dificulta a navegação e a interpretação correta do conteúdo por usuários que utilizam tecnologias assistivas. Para essa finalidade, utilize CSS.

De acordo com as diretrizes WCAG, é fundamental que todas as regiões de uma página web sejam identificadas de forma semântica. Isso significa que cada área da página deve ser marcada com elementos HTML que representem adequadamente seu propósito e tipo de conteúdo. Regiões da Página

Todas as regiões são utilizadas para organizar os conteúdos da página segundo o tipo de conteúdo e não há mais do que uma região do tipo ‘header’/’banner’, ‘main’ ou ‘footer’/’contentinfo’.

A estrutura de uma página web deve ser organizada de forma semântica para garantir acessibilidade adequada. A WCAG recomenda que as regiões principais da página (como <header>, <main> e <footer>) sejam utilizadas uma única vez para evitar confusão. Isso significa que não deve haver mais de um cabeçalho ou rodapé global na página. Essas regiões permitem a correta navegação por tecnologias assistivas, como leitores de tela, e ajudam os usuários a entenderem rapidamente a função de cada parte da página. Outras regiões, como <nav> e <aside>, podem ser usadas conforme necessário para fornecer suporte e organização adicional ao conteúdo.

Abaixo um exemplo da correta utilização destes elementos.

Explicação Exemplo:

  • Cabeçalho (<header>): Usado para definir a área de cabeçalho da página, onde colocamos o título e a navegação principal. Segundo a WCAG, deve haver apenas um <header> global na página para evitar confusão semântica.
  • Navegação (<nav>): A área de navegação permite uma organização clara dos links, garantindo acessibilidade para navegação rápida entre as seções do site.
  • Conteúdo Principal (<main>): A região <main> contém o conteúdo principal da página. Por recomendação da WCAG, deve haver apenas uma região <main> em toda a página.
  • Conteúdo Complementar (<aside>): O elemento <aside> é utilizado para exibir informações adicionais que complementam o conteúdo principal. Ele é semântico e ajuda na organização da página.
  • Rodapé (<footer>): A região <footer> contém informações de rodapé e links relevantes. Assim como o <header>, o rodapé global deve ser único.
